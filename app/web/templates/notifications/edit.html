▸ Guide rapide — comment utiliser les champs ?

Slug

Identifiant unique appele depuis Home Assistant via rest_command.bot_discord. Minuscules + chiffres + underscore uniquement.

Channel Discord

Choisis dans la liste deroulante (channels detectes par le bot ou tes preconfigures dans Parametres) ou colle un ID brut.

Couleur

Click sur la pastille pour une couleur custom, ou utilise un preset enregistre dans Parametres > Couleurs.

Markdown dans le message

Discord supporte le markdown : **gras**, *italique*, __souligne__, ~~barre~~, `code`. Selectionne un mot et clique sur les boutons B / I pour les inserer rapidement.

Champs (fields) — placeholders Home Assistant

Les fields sont affiches en grille dans l'embed (un par carre quand "En grille" est coche, sinon une ligne pleine).

Dans la valeur tu peux melanger du texte et des placeholders qui seront remplaces a l'envoi par les etats Home Assistant :

  • {state:sensor.temp} — etat brut
  • {state:sensor.temp|--} — avec fallback si HA est down
  • {attr:sensor.temp:friendly_name} — un attribut precis
  • {unit:sensor.temp} — l'unite (ex. °C)

Exemple : {state:sensor.temp_salon} {unit:sensor.temp_salon} affichera 21.5 °C. Tu peux utiliser "Inserer une entite" sous le message ou la valeur d'un field pour piocher une entite directement.

Boutons

Supprimer et Snooze sont gerees par le bot. Les boutons d'action appellent un service Home Assistant (ex. light.turn_on sur light.salon).

Timestamp

Quand active, la date d'envoi est ajoutee dans le pied de l'embed (ex. 28 avril 2026 a 14:05).
